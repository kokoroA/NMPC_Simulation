# RTI ＋ 準ニュートン法（TR1/SR2）による NMPC 制御フロー

## Step 1: 状態の取得とシミュレーション誤差（割線条件の準備）

毎周期ヤコビアンを計算する代わりに、「前回の予測モデル」と「実際の非線形モデル」のズレ（誤差）を評価します。

現在の状態 $x_0$ と前回適用した入力 $u_{prev}$ から、実際の非線形ダイナミクスに基づく状態の変化量 $y$ と、入力・状態のステップ幅 $s$ を計算します。これが準ニュートン法の更新の「種」になります。

```cpp
// 非線形モデル f(x, u) を用いて、実際の変化を計算
double x_actual_next[5];
integrate_euler(x_actual_next, x0.data(), u_prev.data(), dt, ...);

// 前回の線形予測モデルとの差分（誤差ベクトル y と、変位ベクトル s）を算出
Eigen::VectorXf y = ... // (非線形の結果 - 線形予測の結果)
Eigen::VectorXf s = ... // (U_+ - U_o, K_+ - K_o)
```

## Step 2: TR1（Two-sided Rank-one）による予測行列 ($S_u$) の直接更新

`calc_jacobian` や `build_prediction_matrices` を使って行列をゼロから構築し直す（$O(N^3)$ の計算）のはやめます。

代わりに、Step 1 で得た誤差情報をもとに、**TR1（ランク 1 更新）**の公式を用いて、前回の予測行列 $S_u$（論文内の文字では $E$ に相当）に少しだけパッチを当てて（差分更新して）新しい予測行列を作ります。

$$
S_u^+ = S_u^{old} + u_1 v_1^T
$$

```cpp
// 論文の式(14)に基づく、予測行列(Su または E)のランク1更新
// 行列の掛け算(O(N^3))ではなく、ベクトル同士の外積(O(N^2))だけで済む
Su = Su_old + u1 * v1.transpose();
```

---

## Step 3: SR2（Symmetric Rank-two）によるヘッセ行列 ($H$) の直接更新

ここが論文の最大のハイライトです。$S_u$ が更新された後、まともに $H = S_u^T Q_{blk} S_u + R_{blk}$ を計算すると重い行列積が発生します。

しかし、$S_u$ が「ランク 1 更新」された事実を利用すると、凝縮されたヘッセ行列 $H$ も「SR2（対称ランク 2 更新）」という軽い足し算だけで直接アップデートできることが数学的に証明されています。

$$
H^+ = H^{old} + \tilde{u}_1 v_1^T + v_1 (\tilde{u}_1 + \beta_1 v_1)^T
$$

```cpp
// 論文の式(16)に基づく、ヘッセ行列のランク2更新
// H = Su.transpose() * Q_blk * Su + R_blk; // ←この重い計算を完全に排除！
H = H_old + u_tilde1 * v1.transpose() + v1 * (u_tilde1 + beta1 * v1).transpose();
```

---

## Step 4: コレスキー分解（L 行列）の低負荷更新（Rank Update）

ヘッセ行列 $H$ が更新されたため、下三角行列 $L$ も更新が必要です。ここで `H.llt()` を最初からやり直す（$O(N^3)$）のではなく、**Cholesky rank update（コレスキー分解のランク更新）**を利用します。

C++ の Eigen ライブラリには、すでに計算済みの LLT オブジェクトに対して低負荷でパッチを当てる `rankUpdate` 関数が用意されています。

```cpp
// 毎周期の完全な再分解（ lltOfH.compute(H) ）は行わない！
// 代わりに、Step 3のランク更新成分（ベクトル）を使って、既存のL行列を直接修正する
lltOfH.rankUpdate(update_vector1, 1.0);   // Rank-1 update を2回行い Rank-2 更新を実現
lltOfH.rankUpdate(update_vector2, -1.0);

Eigen::MatrixXd L = lltOfH.matrixL();    // 超高速に更新されたL行列を取得
```

---

## Step 5: RTI スキームに基づく 1 ステップのみの QP 求解

右辺ベクトル `rhs_full` を作成し、ADMM ソルバーに渡します。

ここで **RTI（Real-Time Iteration）の思想**を適用します。NLP（非線形問題）の最適解が出るまで while ループを何回も回すのではなく、「毎周期、1 回だけ QP を解いて、すぐに出力（モーター PWM）に反映する」というアプローチを取ります。ADMM の初期値には前回の最適解を与えます（ホットスタート）。

---

## このアプローチの圧倒的なメリット

このフローに変更することで、「行列同士の掛け算」や「行列の逆行列・分解のゼロからの計算」がメインループから**完全に消滅**します。すべて「ベクトル同士の掛け算（外積）」と「足し算」に置き換わるため、計算複雑性が $O(N^3)$ から $O(N^2)$ に落ちます。

これにより、Teensy 4.1 や ESP32 といったマイコンの限られた RAM と CPU パワーでも、400 Hz などの超高頻度で NMPC のモデルを動的に更新しながら飛ばすことが可能になります。
